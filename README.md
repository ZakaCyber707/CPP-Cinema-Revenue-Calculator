📊 Logic Overview
This project simulates a point-of-sale system for a cinema or theater using structured arithmetic logic:
Tiered Pricing: The system defines three specific classes with hardcoded values:

Class A: $15 per ticket
Class B: $12 per ticket
Class C: $9 per ticket

Input Collection: Users enter the quantity of tickets sold for each class via the console.
Subtotal Calculation: The program multiplies the quantity by the rate for each class individually.
Grand Total: All subtotals are aggregated into a total variable to display the final revenue.


🛠️ Features
Real-time Processing: Provides immediate feedback for each ticket class as data is entered.
Multi-Variable Management: Demonstrates efficient use of multiple integer variables to track distinct data streams before final summation.


🚀 Execution & Environment Guide
To run this project, you will need a C++ compiler or a dedicated IDE.

1. Recommended IDEs
   Dev-C++
   Visual Studio Code
   Online Compilers: Use OnlineGDB to run the simulation directly in your browser

2. Simple Execution Procedure
(Step 1): Setup the Project
Create a file named Ticket_Sales_Revenue.cpp.
Copy and paste the source code into your editor.

(Step 2): Compile the Code
In Dev-C++: Press F11 to Compile and Run.
In Terminal: Use g++ Ticket_Sales_Revenue.cpp -o RevenueCalc followed by ./RevenueCalc.

(Step 3): Test the Output
Enter the number of tickets sold for each class when prompted to see the total calculated price.
